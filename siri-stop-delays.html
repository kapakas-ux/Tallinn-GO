Real-Time Stop Delays — TLT SIRI API

Endpoint confirmed: https://transport.tallinn.ee/siri-stop-departures.php?stopid=XXXX  ·  Access-Control-Allow-Origin: *  ·  Works from native app.

How It Works

gis.ee queries this endpoint for every stop popup. It returns the real scheduled and expected departure times for each upcoming trip. The delay in seconds is expectedDepartureTime − scheduledDepartureTime.

What this gives us over peatus.ee:
peatus.ee marks most TLT routes as realtime: false, so we only get scheduled times. The SIRI endpoint gives the actual real-time delay in seconds directly from TLT's AVL system — the same data gis.ee shows as "+68s".

1 Find the Exact Response Format

First, check what fields the API actually returns. Open browser DevTools, go to the Network tab, click a bus stop on gis.ee/tallinn, and click the siri-stop-departures.php request → Response tab. Note the field names for:

  • Line number (e.g. lineRef, line, LineRef)
  • Scheduled departure (e.g. aimedDepartureTime, scheduledTime)
  • Expected departure (e.g. expectedDepartureTime, realTime)
  • Delay in seconds (e.g. delay, delaySeconds)
The code below uses the most likely field names. After seeing the real response, update the field names in parseSiriDeparture().

2 Add to src/services/transportService.ts

Add this new function anywhere before fetchDepartures. It fetches the SIRI data and returns a map of lineNumber → delaySeconds for a given stop.

src/services/transportService.ts — add this new function
/**
 * Fetches real-time departure delays from TLT SIRI endpoint.
 * Returns a map: lineRef → { delaySeconds, expectedMinutes }
 * Only covers TLT city buses / trolleys / trams (stopid must be numeric TLT code).
 */
interface SiriDeparture {
  line: string;
  destination: string;
  scheduledTime: number; // unix seconds
  expectedTime: number;  // unix seconds
  delaySeconds: number;
}

async function fetchSiriStopDepartures(stopId: string): Promise<SiriDeparture[]> {
  const url = `https://transport.tallinn.ee/siri-stop-departures.php?stopid=${stopId}`;
  try {
    let rawData: any;

    if (Capacitor.isNativePlatform()) {
      const resp = await CapacitorHttp.get({
        url,
        headers: { 'Accept': 'application/json, text/plain, */*' },
        connectTimeout: 8000,
        readTimeout: 8000,
      });
      if (resp.status !== 200) return [];
      // Log the raw response once so we can verify field names
      console.log('SIRI raw sample:', JSON.stringify(resp.data).substring(0, 500));
      rawData = typeof resp.data === 'string' ? JSON.parse(resp.data) : resp.data;
    } else {
      const resp = await fetch(url);
      if (!resp.ok) return [];
      const text = await resp.text();
      console.log('SIRI raw sample:', text.substring(0, 500));
      rawData = JSON.parse(text);
    }

    // Normalise: handle both array and wrapped { departures: [...] } shapes
    const items: any[] = Array.isArray(rawData)
      ? rawData
      : (rawData.departures || rawData.data || rawData.stopTimes || rawData.MonitoredStopVisit || []);

    return items.map((item: any) => parseSiriDeparture(item)).filter(Boolean) as SiriDeparture[];
  } catch (err) {
    console.warn('fetchSiriStopDepartures failed:', err);
    return [];
  }
}

/**
 * Parses one item from the SIRI response.
 * UPDATE THE FIELD NAMES here once you see the actual response in DevTools.
 */
function parseSiriDeparture(item: any): SiriDeparture | null {
  try {
    // Line number — try all known field name variants
    const line = String(
      item.lineRef || item.LineRef || item.line || item.lineNumber ||
      item.MonitoredVehicleJourney?.LineRef || ''
    ).replace(/^0+/, ''); // strip leading zeros

    // Destination
    const destination = String(
      item.destinationName || item.destination || item.headsign ||
      item.MonitoredVehicleJourney?.DestinationName || ''
    );

    // Scheduled departure — ISO string or unix seconds
    const scheduledRaw =
      item.aimedDepartureTime || item.scheduledDepartureTime || item.scheduledTime ||
      item.AimedDepartureTime || item.MonitoredVehicleJourney?.MonitoredCall?.AimedDepartureTime;

    // Expected (real-time) departure
    const expectedRaw =
      item.expectedDepartureTime || item.realDepartureTime || item.realTime ||
      item.ExpectedDepartureTime || item.MonitoredVehicleJourney?.MonitoredCall?.ExpectedDepartureTime;

    if (!scheduledRaw || !line) return null;

    const toSeconds = (v: any): number => {
      if (typeof v === 'number') return v > 1e9 ? v / 1000 : v; // ms or s
      return Math.floor(new Date(v).getTime() / 1000);
    };

    const scheduledTime = toSeconds(scheduledRaw);
    const expectedTime = expectedRaw ? toSeconds(expectedRaw) : scheduledTime;

    // Delay may be provided directly in the response as well
    const directDelay =
      item.delay ?? item.delaySeconds ?? item.delayInSeconds ?? item.Delay;
    const delaySeconds = directDelay !== undefined
      ? Number(directDelay)
      : (expectedTime - scheduledTime);

    return { line, destination, scheduledTime, expectedTime, delaySeconds };
  } catch {
    return null;
  }
}

3 Update fetchDepartures() to merge SIRI data

Find the existing fetchDepartures function. After the peatus.ee call returns the arrivals array, add the SIRI merge block shown below.

src/services/transportService.ts — inside fetchDepartures(), after arrivals are built
// ─── EXISTING code (find this block) ─────────────────────────────────────────
export async function fetchDepartures(stopId: string, siriId?: string, time?: string): Promise<Arrival[]> {
  try {
    // ... existing peatus.ee GraphQL query ...
    // ... existing arrivals array is built ...

    arrivals.sort((a, b) => a.minutes - b.minutes);
    // ... existing vehicleIndex assignment ...

    // ─── ADD THIS BLOCK right before the final return ─────────────────────

    // Overlay real-time delays from TLT SIRI endpoint.
    // Only applies to stops that have a numeric TLT stop code (siriId).
    // Non-TLT stops (regional buses) are unaffected.
    const tltStopCode = siriId || stopId;
    if (/^\d+$/.test(tltStopCode)) {
      const siriDeps = await fetchSiriStopDepartures(tltStopCode);
      if (siriDeps.length > 0) {
        const nowSeconds = Math.floor(Date.now() / 1000);
        arrivals = arrivals.map(arrival => {
          // Match by line number; if multiple trips, match closest scheduled time
          const candidates = siriDeps.filter(s =>
            s.line === arrival.line ||
            s.line === arrival.line.replace(/^0+/, '')
          );
          if (candidates.length === 0) return arrival;

          // Pick SIRI entry whose scheduled time is closest to this arrival's minutes
          const arrivalScheduledSeconds = nowSeconds + arrival.minutes * 60;
          const best = candidates.reduce((a, b) =>
            Math.abs(a.scheduledTime - arrivalScheduledSeconds) <
            Math.abs(b.scheduledTime - arrivalScheduledSeconds) ? a : b
          );

          // Only use SIRI if the match is within ±3 minutes
          if (Math.abs(best.scheduledTime - arrivalScheduledSeconds) > 180) return arrival;

          const realMinutes = Math.max(0, Math.floor((best.expectedTime - nowSeconds) / 60));
          const delayMin  = Math.round(best.delaySeconds / 60);
          const status: Arrival['status'] = best.delaySeconds > 60 ? 'delayed'
                                                : best.delaySeconds < -30 ? 'on-time'
                                                : 'on-time';

          return {
            ...arrival,
            minutes: realMinutes,
            status,
            // Show delay badge in UI: "+1 min" or "-30s"
            info: best.delaySeconds > 30
              ? `+${best.delaySeconds}s`
              : best.delaySeconds < -30 ? `${best.delaySeconds}s` : undefined,
          };
        });
        console.log(`SIRI overlay applied: ${siriDeps.length} real-time entries for stop ${tltStopCode}`);
      }
    }

    // ─── END of new block ─────────────────────────────────────────────────

    return arrivals.slice(0, time === '0' ? 50 : 10);
  } catch (error) { ... }
}
Note: The arrivals variable must be declared with let not const for the arrivals = arrivals.map(…) line to work. Check the existing declaration and change it if needed.
